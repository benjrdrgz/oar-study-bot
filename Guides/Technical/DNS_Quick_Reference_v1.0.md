## DNS RECORDS TO ADD (For Each Outreach Domain)

### MX Records (Google Workspace Email)
Add all 5 records with these values:

| Priority | Value |
|----------|-------|
| 5 | gmail-smtp-in.l.google.com. |
| 10 | alt1.gmail-smtp-in.l.google.com. |
| 20 | alt2.gmail-smtp-in.l.google.com. |
| 30 | alt3.gmail-smtp-in.l.google.com. |
| 40 | alt4.gmail-smtp-in.l.google.com. |

### SPF Record
| Type | Name | Value |
|------|------|-------|
| TXT | @ | v=spf1 include:_spf.google.com ~all |

### DKIM Record
Generated by Google Workspace Admin Console. Add the CNAME records Google provides:

| Type | Name | Value |
|------|------|-------|
| CNAME | google._domainkey | google.c._domainkey.yourdomain.com. |

(Exact value provided by Google Admin Console → Security → Authenticate Email)

### DMARC Record
| Type | Name | Value |
|------|------|-------|
| TXT | _dmarc | v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com |

Replace `yourdomain.com` with your actual domain (e.g., localwebsolutions.com)

## COMMON MISTAKES TO AVOID

| Mistake | Why It's Bad | Fix |
|---------|------------|-----|
| Skipping warm-up | Emails go straight to spam | Wait full 14 days |
| Multiple SPF records | Only 1 SPF per domain allowed | Delete old one, keep only Google's |
| Using `-all` instead of `~all` | Hard fail blocks legitimate mail | Use `~all` (soft fail) |
| Wrong DKIM CNAME values | Google can't verify | Copy exact values from Admin Console |
| Not waiting for DKIM activation | Status stuck on "Pending" | Wait 24-48 hours for Google to verify |
| Forgetting `_dmarc` in DMARC name | Record isn't found | Name MUST be `_dmarc` (with underscore) |
| Not using second domain | Sender reputation gets damaged | Rotate domains every 2-4 weeks |
| Sending too many emails too fast | Gmail flags as spam | Keep under 150-200/day per account |
| Missing trailing dots on DNS values | Record doesn't resolve | Most registrars add automatically, but check |

## EMERGENCY TROUBLESHOOTING

**Emails going to spam?**
→ See full guide Section 6, subsection "My Emails Are Going to Spam"

**Can't connect email to Instantly?**
→ Verify MX records on mxtoolbox.com (should show Google's MX records)
→ Make sure you used Google Workspace email, not Gmail

**SSL certificate shows "Not Secure"?**
→ Wait 24 hours (Let's Encrypt provision time)
→ In Netlify: Delete domain → Re-add domain → Wait 30 minutes

**Subdomain won't resolve?**
→ Check registrar DNS (verify CNAME is correct)
→ Check mxtoolbox.com for CNAME propagation
→ Wait 30 minutes for DNS propagation
→ Clear browser cache (Cmd+Shift+R)
